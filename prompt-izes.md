Você é meu desenvolvedor front-end sênior. Crie o APP da IZES, uma plataforma de decisão agrícola assistida para pequenos produtores e microprodutores.

Use as skills disponíveis de forma eficiente:
- brandkit: manter identidade visual consistente da IZES
- design-taste-frontend: criar visual moderno, profissional e com bom senso estético
- high-end-visual-design: evitar aparência genérica de IA
- human-design: deixar a interface mais natural, humana e acessível
- image-to-code: caso existam referências visuais, transformar em layout fiel
- frontend-mobile e imagegen-frontend-mobile: priorizar responsividade mobile
- imagegen-frontend-web: criar versão web responsiva
- gpt-taste e impactful: melhorar textos, hierarquia visual e impacto
- stitch-design-taste: unir as decisões visuais de forma coerente
- minimalist-ui: manter simplicidade e clareza
- design-existing-projects: respeitar a estrutura do projeto existente
- full-output-enforcement: entregar arquivos completos, sem trechos soltos

Objetivo:
Criar um app funcional e visualmente forte para a IZES, com foco em transformar dados do campo em decisões práticas.

Público:
Pequenos produtores rurais, microprodutores, cooperativas, técnicos agrícolas e parceiros do agro.

Tom visual:
Moderno, confiável, limpo, agrícola, tecnológico e acessível. Evite layout padrão de IA, cards genéricos demais, excesso de gradientes e aparência de template. Quero algo com cara de startup agro real.

Paleta sugerida:
Use tons de verde, terra, off-white e cinza escuro. A interface deve passar confiança, simplicidade e inteligência.

Funcionalidades principais:
1. Dashboard inicial com resumo da propriedade
2. Alertas inteligentes: “Ação urgente”, “Atenção” e “Tudo certo”
3. Tela de recomendações práticas dizendo:
   - o que fazer
   - onde agir
   - prioridade
   - motivo da recomendação
4. Mapa ou visualização dos talhões/áreas da propriedade
5. Monitoramento ambiental:
   - umidade do solo
   - temperatura
   - chuva
   - risco de praga/doença
6. Histórico de decisões e ações realizadas
7. Tela de perfil da propriedade
8. Tela simples de onboarding para produtor cadastrar cultura, área e objetivo

Frase central da marca:
“Dados sozinhos não mudam o agro. Decisões mudam. E o IZES existe para transformar informação em ação no momento certo.”

Diferenciais que precisam aparecer claramente:
- acessibilidade
- simplicidade
- foco no pequeno produtor
- transformação de dados em decisões práticas

Regras para economizar tokens:
1. Antes de codar, faça um plano curto com no máximo 8 etapas.
2. Não explique demais.
3. Não gere documentação longa.
4. Não repita código desnecessariamente.
5. Altere somente os arquivos necessários.
6. Se o projeto já existir, primeiro analise a estrutura e reaproveite componentes.
7. Entregue uma implementação incremental.
8. Evite criar muitas dependências.
9. Use componentes reutilizáveis.
10. Ao final, apenas liste os arquivos alterados e como rodar o projeto.

Regras de layout:
1. Nada de visual genérico estilo “SaaS AI dashboard” comum.
2. Use hierarquia visual forte.
3. Crie microtextos humanos e diretos.
4. Use cards úteis, mas com variação visual.
5. Priorize clareza para alguém do campo.
6. Mobile first.
7. Interface simples o suficiente para um pequeno produtor entender sem treinamento.
8. O app deve parecer premium, mas acessível.

Entrega esperada:
- Criar as telas principais do app IZES
- Implementar navegação entre telas
- Criar componentes reutilizáveis
- Criar dados simulados realistas
- Deixar responsivo para mobile e desktop
- Garantir boa organização de pastas
- Não usar imagens externas obrigatórias
- Usar ícones simples se já houver biblioteca disponível

Comece analisando o projeto atual. Depois proponha um plano curto e implemente a primeira versão funcional do APP da IZES.
